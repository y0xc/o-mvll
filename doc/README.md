# Building the O-MVLL documentation

The API reference is generated by [Sphinx](https://www.sphinx-doc.org/) using
the [`autodoc`](https://www.sphinx-doc.org/en/master/usage/extensions/autodoc.html)
extension. **The single source of truth is the C++ docstrings** embedded in the
pybind11 bindings:

- `src/core/python/PyConfig.cpp` — `omvll.OMVLLConfig`, the `omvll.Phase` /
  `omvll.Pass` enums, and the `omvll.ObfuscationConfig` callbacks.
- `src/core/python/pyobf_opt.cpp` — the `*Opt` option classes.
- `src/core/python/pyllvm.cpp` — `Module`, `Function`, `Struct`,
  `GlobalVariable`.

The `.rst` files under `doc/src/` are thin: they only provide section structure
and `.. autoclass::` / `.. autodoc` directives that pull the docstrings out of
the compiled module. **To document a class/method/attribute, edit its C++
docstring — not the `.rst`.**

A few items that are not class members stay manual in the `.rst` files: the
`omvll.config` instance attribute and the configuration `Template`
(`doc/src/config.rst`, `doc/src/obfuscation.rst`) and the version attributes
(`doc/src/versioning.rst`).

## Why a "standalone" module is needed

`autodoc` works by importing the `omvll` Python module and introspecting its
docstrings. O-MVLL normally builds as an LLVM **pass plugin** (a shared library
loaded by `opt`/`clang`), which is *not* importable from Python.

Building with `-DOMVLL_PY_STANDALONE=1` instead produces an importable Python
extension module named `omvll`. `doc/conf.py` imports it directly, falling back
to the directory given by the `OMVLL_STANDALONE_DIR` environment variable (or
`PYTHONPATH`).

## Option A — Linux / Docker (CI path)

This is what CI uses. See `scripts/docker/doc.sh`; in short:

```sh
cmake -GNinja -S src -B /standalone -DOMVLL_PY_STANDALONE=1 -DCMAKE_BUILD_TYPE=Release ...
ninja -C /standalone                       # → /standalone/omvll.so
OMVLL_STANDALONE_DIR=/standalone sphinx-build -b html doc doc/_build/html
```

## Option B — Local macOS

Requires the prebuilt dependency tree (LLVM 21.1.6, pybind11, spdlog) and
**Python 3.10** for both the build and Sphinx. The bundled pybind11 (2.10.3)
does not compile against Python 3.14, so do not use a newer interpreter.

```sh
# 1. Sphinx environment (Python 3.10)
python3.10 -m venv /tmp/omvll-doc-venv
/tmp/omvll-doc-venv/bin/pip install sphinx furo

# 2. Build the standalone module
# Dependencies for version X.Y.Z of O-MVLL can be downloaded via
# https://open-obfuscator.build38.io/static/omvll-vX.Y.Z-macos-deps.tar
DEPS=~/Workspace/omvll-v1.9.1-macos-deps
LLVM=$DEPS/LLVM-21.1.6-arm64-Darwin
PY=/opt/homebrew/opt/python@3.10

cmake -GNinja -S src -B build \
  -DOMVLL_PY_STANDALONE=1 \
  -DCMAKE_BUILD_TYPE=Release \
  -DPython3_EXECUTABLE=$PY/bin/python3.10 \
  -DPython3_INCLUDE_DIR=$PY/Frameworks/Python.framework/Versions/3.10/include/python3.10 \
  -DPython3_LIBRARY=$PY/Frameworks/Python.framework/Versions/3.10/lib/libpython3.10.dylib \
  -Dpybind11_DIR=$DEPS/share/cmake/pybind11 \
  -Dspdlog_DIR=$DEPS/lib/cmake/spdlog \
  -DLLVM_DIR=$LLVM/lib/cmake/llvm \
  -DClang_DIR=$LLVM/lib/cmake/clang
ninja -C build OMVLL                            # → build/omvll.so

# 3. Generate the HTML
OMVLL_STANDALONE_DIR=$(pwd)/build \
  /tmp/omvll-doc-venv/bin/python -m sphinx -b html doc doc/_build/html
```

On macOS the standalone target is emitted as `omvll.so` (not `omvll.dylib`) and
keeps `PyInit_omvll` exported, so it is importable directly — see the
`OMVLL_PY_STANDALONE` / `APPLE` handling in `src/CMakeLists.txt`.

## Notes on the rendered output

`autodoc` renders pybind11 objects in its own style, which differs cosmetically
from hand-written directives:

- Read-only attributes appear as `property name` with a separate **Type:**
  field rather than `name : type`.
- Methods and constructors show their full signature, including `self`,
  parameter types, and the return type.
- Enums (`Phase`, `Pass`) carry a pybind11-generated **Members:** list.

Build with `-W --keep-going` to treat unresolved cross-references and other
warnings as errors:

```sh
OMVLL_STANDALONE_DIR=$(pwd)/build \
  python -m sphinx -b html -W --keep-going doc doc/_build/html
```
